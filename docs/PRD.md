# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述
### 背景與動機
現代人生活繁忙，許多喜愛料理的民眾常在網路各處看到想嘗試的食譜，卻總是散落在不同平台或以截圖形式存在手機裡，不易整理與尋找。為了幫助使用者更有效率地管理與發現料理靈感，本系統旨在提供一個專屬的「食譜收藏夾」，整合所有的食譜資源。

### 目標用戶
對料理有興趣、有煮飯習慣，或想要建立個人食譜資料庫的群眾。

### 核心價值主張
提供一個簡潔、容易分類與搜尋的食譜集中地，並透過推薦系統激發用戶的料理靈感，讓找食譜、做菜變得更輕鬆有趣。

## 2. 功能需求
以下為本系統的主要功能與對應的使用者故事：

1. **食譜收藏與分類**
   - 作為 **熱愛料理的用戶**，我希望 **可以將網路上的食譜連結儲存下來，並設定自訂分類與標籤（如：減脂、日式、快速晚餐）**，以便 **我日後能快速找到特定種類的料理**。

2. **食譜搜尋與篩選**
   - 作為 **猶豫晚餐吃什麼的用戶**，我希望 **能透過關鍵字、食材或分類標籤進行多重條件搜尋與篩選**，以便 **我能根據現有食材或想吃的口味快速找出合適的食譜**。

3. **個人化推薦**
   - 作為 **不知今天該煮什麼的用戶**，我希望 **系統能根據我過往的收藏習慣或季節食材，自動推薦可能感興趣的食譜**，以便 **我能獲得新的料理靈感**。

4. **購物清單自動生成**
   - 作為 **準備去超市採買的用戶**，我希望 **能一鍵將欲製作的食譜內的食材明細轉換為購物清單**，以便 **我可以直接對照手機採買，不會遺漏且方便管理**。

5. **使用者上傳與評價系統**
   - 作為 **喜歡分享與紀錄料理心得的用戶**，我希望 **能上傳自創食譜，並能對自己做過或他人分享的食譜給予星級評價與留言**，以便 **我能記錄每次料理的優缺點，也能幫助其他用戶參考**。

## 3. 非功能需求
### 技術限制
- 後端框架：Flask
- 前端技術：Jinja2 (與 HTML/CSS/JavaScript 搭配)
- 資料庫：SQLite

### 效能與安全考量
- **響應速度**：頁面載入時間與搜尋功能應即時響應，保持流暢的使用者體驗。
- **安全性**：實作基礎的用戶身份驗證與授權機制，保護使用者的密碼與私人資料，並防範常見的 XSS 與 CSRF 攻擊。
- **行動裝置友善**：前端介面需具備 RWD (響應式設計)，讓用戶在廚房透過手機或平板觀看食譜時也能享受舒適的閱讀體驗。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (核心必備)**
  - 使用者註冊與登入
  - 建立(上傳)、檢視、編輯、刪除食譜（CRUD）
  - 基本的食譜分類或是標籤功能
  - 關鍵字搜尋食譜

- **Should Have (重要功能)**
  - 食材篩選與多條件搜尋
  - 食譜評價與簡單的留言功能
  - 從食譜自動生成購物清單

- **Nice to Have (錦上添花)**
  - 個人化推薦機制
  - 進階社群互動與外部食譜匯入
